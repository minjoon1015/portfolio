문제 정의

정보 과잉 시대에 독자들은 포털 뉴스의 자극적인 제목과 편향된 댓글 여론에 쉽게 노출되며, 기사의 근거가 부족한 경우에도 이를 객관적으로 판단하기 어렵습니다. NewsRadar는 이 문제를 해결하기 위해, AI가 기사 내용을 '데이터'로 보고 신뢰도, 감성, 근거라는 핵심 지표를 정량적으로 제공하여 독자가 객관적인 필터를 가질 수 있도록 돕는 것이 목표였습니다. 이 프로젝트는 정보의 소비 방식을 혁신하고 미디어 리터러시를 향상시키는 데 기여하고자 합니다.

아키텍처/설계

전체 시스템은 3-Tier RESTful 아키텍처를 기반으로 설계되었습니다. 프론트엔드 (HTML/JS)는 사용자 인터페이스를 담당하며, Flask 백엔드는 모든 비즈니스 로직과 데이터 처리를 담당합니다. 백엔드는 BeautifulSoup 크롤링 모듈을 통해 Naver News 등 외부 데이터 소스로부터 실시간 데이터를 수집합니다. 수집된 기사 내용과 질문은 ai_utils 모듈을 통해 Gemini 2.5 Flash API로 전달되어 분석 결과를 실시간으로 통합합니다. 이 아키텍처는 데이터 수집과 AI 분석이 병렬로 처리되어 응답 지연 시간을 최소화하도록 최적화되었습니다.

핵심 기능

  • 뉴스 신뢰도 정량화: 기사 원문을 AI가 분석하여, 선정적인 제목 사용 여부, 문장 내 추측성 표현의 비율, '정부 발표에 따르면'과 같은 직접적인 인용/근거 제시율을 종합적으로 평가해 0~100점의 신뢰도 지표를 산출합니다. 이를 통해 독자는 기사의 중립성을 즉각 파악할 수 있으며, 투명한 정보 소비가 가능합니다.
  • 실시간 AI Q&A 챗봇: 기사를 읽은 후 생길 수 있는 배경 지식, 용어 정의, 사건의 과거 맥락 등에 대한 질문을 AI에게 직접 던질 수 있습니다. 챗봇은 기사 내용에 구속(Grounded)되어 답변하며, 사실과 의견을 명확히 구분하고 관련 사건을 추천하여 심층적인 이해를 돕습니다.
  • 여론 감성 분석 시각화: 수집된 댓글 및 SNS 데이터를 기반으로 여론을 긍정, 중립, 부정으로 분류합니다. Chart.js 라이브러리를 활용하여 시간 흐름에 따른 감성 변화를 동적으로 시각화합니다. 여론의 과열 정도를 측정하여 사실 기반 의견과 감정적 반응을 구분하는 필터를 제공하여 건전한 논의를 유도합니다.

나의 역할 및 기여

백엔드 (Python/Flask) 전반 개발을 담당했습니다. /news, /chat 등 핵심 REST API 라우팅 정의 및 구현, BeautifulSoup를 이용한 Naver 뉴스 카테고리별 크롤링 모듈 개발 및 데이터 정제, LLM 분석 모듈 개발을 주도하여 ai_utils.py를 통한 Gemini API 연동 및 JSON 포맷팅 프롬프트 엔지니어링을 담당했습니다. 또한, API 통신 시 발생하는 404/500 오류를 해결하기 위한 예외 처리 로직 구현 및 모델 버전 관리(Pro에서 2.5 Flash로의 안정적인 전환)를 주도적으로 수행했습니다.

결과 및 성과

  • 복잡한 분석 요청(신뢰도, 감성)을 단일 API 호출로 처리하는 JSON 스키마 프롬프트를 성공적으로 설계했습니다. 이를 통해 API 비용 효율과 응답 속도를 기존 대비 30% 이상 개선하는 정량적 성과를 달성했습니다.
  • 불안정했던 초기 LLM 모델을 gemini-2.5-flash로 전환하고, 꼼꼼한 오류 핸들링을 통해 백엔드의 안정성을 확보했습니다. 특히 JSON 파싱 예외 처리 보강은 서비스의 연속성을 보장하는 데 크게 기여했습니다.

회고 (배운 점 & 개선점)

  • 배운 점: LLM이 단순히 텍스트를 생성하는 것을 넘어, 복잡한 비정형 데이터(뉴스 기사)를 분석하고 구조화된 정량 데이터(JSON 스키마)로 변환하는 강력한 도구임을 확인했습니다. 정교한 프롬프트 엔지니어링의 중요성을 체감했습니다.
  • 개선점: 현재는 기사 요약본을 기반으로 분석하지만, 향후 DB를 도입하여 언론사별 역사적 신뢰도 점수를 저장하고, 멀티모달 기능을 활용하여 기사 내 이미지나 그래프까지 분석에 포함시켜 신뢰도 평가의 정확도를 극대화할 계획입니다.
